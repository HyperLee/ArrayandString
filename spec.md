# Array / String 教學專案開發規格書

## 1. 文件目的

本文件用於定義 `ArrayandString` 專案的開發目標、內容範圍、交付物、功能需求與驗收標準，作為後續實作與文件撰寫的依據。

本階段目標為完成「開發規格書」，暫不進行程式功能開發。

---

## 2. 專案背景

在電腦科學中，`Array` 與 `String` 是最基礎且最常用的資料結構之一。字串通常可視為字元序列，而陣列則是儲存同型別資料的連續集合。兩者在資料表示、操作方式、記憶體特性與實務應用上皆有高度關聯，因此非常適合作為初學者理解資料結構與演算法的起點。

目前專案為一個以 `.NET 10` 為基礎的 Console Application。後續將以此專案為載體，建立一份完整的 Array / String 教學內容，包含：

- 基本概念介紹
- 特性與差異說明
- C# 實際程式範例
- 程式碼註解解說
- README 教學文件
- LeetCode 練習題目整理

---

## 3. 專案定位

本專案定位為「教學型範例專案」，不是商業系統，也不是演算法題解庫。

主要用途如下：

- 提供初學者理解 Array 與 String 的核心觀念
- 提供以 C# 撰寫的可執行範例
- 提供文件化整理，方便閱讀與複習
- 提供後續演算法練習的學習路徑

---

## 4. 開發目標

本專案後續開發應達成以下目標：

1. 建立一份完整且有條理的 `README.md`，系統性介紹 Array 與 String。
2. 建立對應的 `.cs` 範例程式碼，展示核心操作與實際使用情境。
3. 讓程式碼具備教學可讀性，包含必要註解與清楚命名。
4. 以 Array / String 為主軸，補充常見應用、陷阱與學習重點。
5. 整理一批 LeetCode 練習題，讓使用者可延伸練習與實作。

---

## 5. 目標使用者

本專案的主要讀者與使用者如下：

- 剛接觸程式設計與資料結構的初學者
- 想用 C# 理解 Array 與 String 的學習者
- 準備演算法面試或 LeetCode 練習的使用者
- 想快速複習 Array / String 基礎觀念的開發者

## 9. 功能需求

以下需求為後續開發時應滿足的功能規格。

### 9.1 README 文件需求

`README.md` 必須包含以下章節：

1. 專案簡介
2. Array 基本概念
3. String 基本概念
4. Array 與 String 的共同點與差異
5. C# 中的 Array 使用方式
6. C# 中的 String 使用方式
7. 常見使用情境
8. 範例程式碼說明
9. 常見錯誤與注意事項
10. LeetCode 練習建議
11. 總結

`README.md` 撰寫要求如下：

- 內容需使用繁體中文
- 結構需清楚分段，適合初學者閱讀
- 理論說明需搭配程式範例
- 程式範例需與專案中的 `.cs` 程式碼相對應
- 不可只有定義，需補充「何時使用」與「為何這樣設計」

### 9.2 C# 範例程式需求

程式範例需至少涵蓋以下主題：

#### Array 範例

- 一維陣列宣告與初始化
- 使用索引讀取元素
- 修改指定索引的值
- 使用迴圈走訪陣列
- 陣列長度存取
- 二維陣列或多維陣列基礎示範
- 常見方法或實用操作示範

#### String 範例

- 字串宣告與初始化
- 使用索引讀取字元
- 字串長度存取
- 字串串接
- 擷取部分字串
- 搜尋、取代、分割等常見操作
- 字串不可變性的示範

#### Array 與 String 比較範例

- 可變與不可變的差異
- 元素型別差異
- 記憶體概念的入門說明
- 典型使用時機比較

### 9.3 程式註解需求

所有教學型範例程式需加上適量註解，註解應符合以下原則：

- 用於說明程式目的與重要觀念
- 解釋該段程式想展示的概念
- 避免過度冗長或逐行翻譯型註解
- 註解內容需與 README 說明一致

### 9.4 Console 呈現需求

若以 Console App 呈現，內容至少應具備以下特性：

- 有清楚的章節分隔
- 每段輸出能對應一個學習主題
- 範例輸出需可讀且有說明性
- 使用者執行後可理解目前示範的是哪個概念

可接受的呈現方式如下：

- 依主題順序直接輸出完整教學內容
- 以簡單選單分流到不同主題

最終採用哪一種方式，可在實作階段決定，但必須保持教學可讀性。

### 9.5 LeetCode 練習題需求

需在 README 中整理至少一個獨立章節，提供 Array / String 相關練習題建議，內容至少包含：

- 題目名稱
- 題目類型
- 難度分級
- 對應練習重點
- 建議練習順序

建議涵蓋：

- Easy
- Medium
- 進階延伸題

不強制提供完整題解，但需說明該題與 Array / String 哪些觀念相關。

## 16. 風險與注意事項

後續開發時需注意以下風險：

- 若內容只堆疊概念，容易失去教學實用性
- 若程式碼範例太少，文件會偏向理論而不易理解
- 若程式碼註解過多，可能造成閱讀負擔
- 若 README 與程式碼不同步，會降低專案可信度
- 若 LeetCode 題目只列名稱而無分類說明，學習價值會不足
